Context & Motivation

Problem Statement

⚠ The Challenge

Steel manufacturing plants produce thousands of plates per shift. Defective plates reaching customers result in costly recalls, safety risks, and reputational damage.

📊 Manual Inspection Limits

Manual visual inspection is slow, subjective, and error-prone. Operators struggle with 7 distinct defect types sharing overlapping visual features.

🎯 Research Goal

Build an Intelligent Decision Support System that automatically classifies steel plate defects, quantifies uncertainty, explains decisions, and supports operators with actionable guidance.

🗄 Dataset

UCI Steel Plates Faults (ID 198) — 1,941 plates, 27 sensor features, 7 mutually exclusive defect classes. Severe class imbalance: 12:1 ratio (Other_Faults=673 vs Dirtiness=55).

📊 Phase 1 — Exploratory Data Analysis

Understanding the Dataset

Key Findings

  • 📌 1,941 steel plate samples with 27 numeric sensor readings from conveyor imaging

  • ⚠️ Severe class imbalance (12:1) — Other_Faults (673) dominates; Dirtiness (55) is smallest

  • 🔗 Multi-collinear features — X_Minimum/X_Maximum, perimeter pairs show high correlation

  • 📐 All features numeric, no missing values — two binary steel-type flags (A300/A400)

  • 🎯 Mutually exclusive labels — every plate has exactly one defect class (row sums to 1)

⚠ Class Imbalance Impact

Standard accuracy is misleading — a model that always predicts Other_Faults gets 34.7% accuracy for "free". This drove the choice of Macro OVR AUC as the primary metric, which treats all classes equally.

Why Stacking Ensemble?

Although Soft Voting has marginally higher AUC (0.9626), the Stacking Ensemble was chosen for its structured meta-learner — enabling SHAP explainability via the RF base estimator while maintaining competitive performance and better Macro F1 (0.794).

Actionable Intelligence

5 Business Recommendations

01 — Automation
Auto pass/reject on High-confidence plates
Deploy automated actuators on the High-tier queue (≥70% confidence). Removes manual bottleneck for the majority of production.
Evidence: 65.8% of plates qualify at 88.3% accuracy → estimated 42% reduction in operator inspection time
02 — Sensor Maintenance
Monthly calibration of Length_of_Conveyer
Length_of_Conveyer is the #1 SHAP driver (mean |SHAP| = 0.019). Sensor drift will directly degrade model predictions.
Evidence: Top SHAP feature across all 7 defect classes — calibration SLA should match model retraining frequency
03 — Safety Net
Mandatory human review for Other_Faults
Other_Faults is the model's weakest class (AUC 0.890). All predictions in this class should bypass automation regardless of confidence tier.
Evidence: AUC 0.890 = lowest of all classes; confused with Bumps (0.17) and Pastry (0.08)
04 — Active Learning
Monthly retraining on Medium-tier plates
Medium-tier plates (40–70% confidence, 26.7% of volume) are the most informative for model improvement. Operator-confirmed labels from this tier should feed the next retraining cycle.
Evidence: ~520 Medium-tier plates per 1,941 batch → rich training signal with minimal annotation cost
05 — Camera Diagnostics
Alert on consecutive Stains/Dirtiness predictions
Consecutive Stains or Dirtiness predictions on otherwise-clean batches likely indicate a dirty camera lens rather than true defects. Trigger a cleaning alert automatically.
Evidence: Stains (AUC 0.961) & Dirtiness (AUC 0.991) are environment-sensitive — surface contamination on lens mimics plate contamination
